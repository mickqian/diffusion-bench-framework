Where the speed comes from

Why it wins

Kernels that already shipped. A diffusion transformer runs the same block hundreds of times per request. The battle-tested attention and GEMM kernels behind SGLang's LLM runtime carry straight over — FlashAttention-3 on Hopper, fused norm and residual paths, no warm-up tax on every step.

A native pipeline, not a wrapper. SGLang serves FLUX and Qwen-Image through purpose-built pipelines rather than a generic diffusers fallback. That removes per-step Python overhead and lets the scheduler batch, parallelise and cache where it helps — and skip it cleanly where the comparison must stay cache-free.

Compile where it pays. Regional torch.compile over the repeated DiT block keeps cold starts sane while still fusing the hot path. The numbers here are steady-state and cache-free, so what you read is the kernel, not a cache hit.

Honest about the gaps. Where a framework has no serving path for a model — or simply can't load it — the ledger says so plainly. TensorRT-LLM VisualGen is image-only; LightX2V tracks a fixed model set. No silent blanks.

Method

Single end-to-end request, steady-state (one-time compile excluded), no response cache, no Cache-DiT, no temporal cache. Same H200 SKU and GPU count per row.
