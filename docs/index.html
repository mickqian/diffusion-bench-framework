Native image and video serving

SGLang-Diffusion

A native serving stack for image and video generation. The main performance gains come from the inference system around the model: batching, disaggregation, memory residency, parallel execution, kernel paths, and carefully measured lossy acceleration.

Where the gains come from

The clean way to discuss SGLang-Diffusion is to separate output-preserving inference-stack gains from lossy optimizations that trade computation for measured quality drift.

Lossless inference-stack gains

Preserve the generation recipe. Improve throughput, latency, and memory by changing how the serving stack schedules, places, parallelizes, and executes the pipeline.

Lossy acceleration frontier

Change the amount or precision of computation. Report speed only together with quality drift, artifact checks, and an explicit acceptance threshold.

What carries across image and video serving

The useful lesson is not a benchmark trick. It is how to turn diffusion pipelines into a serving system with explicit request lifecycle, stage boundaries, memory policy, parallelism, and quality-aware acceleration.

Native serving creates the optimization surface

Batching, scheduling, cancellation, observability, and output handling need to live in the server, not in one-off inference scripts.

Stage boundaries are scaling boundaries

Encoders, denoisers, and decoders bottleneck differently. Treating them as roles makes disaggregation and placement meaningful.

Memory policy is performance policy

Keeping hot denoising paths resident while offloading colder encoder work can improve both model fit and latency stability.

Parallelism has model-specific break-even points

CFG, TP, SP, and Ulysses should be selected by model shape and hardware behavior, not treated as universally faster toggles.
